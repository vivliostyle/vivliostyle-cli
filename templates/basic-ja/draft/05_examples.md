# 実践例

## 例1：技術文書

この例では、プロフェッショナルな技術文書の作り方を紹介します。

### ドキュメント構成

典型的な技術文書には以下が含まれます：

1. **はじめに**: トピックの概要
2. **インストール**: セットアップ手順
3. **使い方**: ソフトウェアやAPIの使い方
4. **リファレンス**: 詳細なAPIやコマンドのドキュメント
5. **トラブルシューティング**: よくある問題と解決策

### コードドキュメント

APIを明確な例とともにドキュメント化します：

```typescript:api-client.ts
interface UserProfile {
  id: string;
  name: string;
  email: string;
}

async function fetchUser(userId: string): Promise<UserProfile> {
  const response = await fetch(`/api/users/${userId}`);
  if (!response.ok) {
    throw new Error('Failed to fetch user');
  }
  return response.json();
}
```

### コマンドリファレンス

コマンドラインツールを構造化されたテーブルでドキュメント化します：

| コマンド    | 説明                   | オプション               |
| ----------- | ---------------------- | ------------------------ |
| `init`      | 新しいプロジェクトを初期化 | `--template`, `--theme` |
| `build`     | 出力ファイルを生成         | `-o`, `-s`, `--theme`   |
| `preview`   | プレビューサーバーを起動   | `--quick`, `--port`     |

## 例2：学術論文

学術出版物には正確なフォーマットと引用のサポートが必要です。

### 数学的証明

**定理1.** _任意の実数 $x$ に対して、等式 $e^{i\pi} + 1 = 0$ が成り立つ。_

**証明.** オイラーの公式 $e^{i\theta} = \cos\theta + i\sin\theta$ を用いて、$\theta = \pi$ を代入すると：

$$
e^{i\pi} = \cos\pi + i\sin\pi = -1 + 0i = -1
$$

したがって：

$$
e^{i\pi} + 1 = -1 + 1 = 0
$$

これで証明が完了しました。$\square$

## 例6：混合メディア

### コードと解説

エラーハンドリングの完全な例を示します：

```js:error-handler.js
class APIError extends Error {
  constructor(message, statusCode) {
    super(message);
    this.statusCode = statusCode;
    this.name = 'APIError';
  }
}

function handleError(error) {
  if (error instanceof APIError) {
    console.error(`API Error ${error.statusCode}: ${error.message}`);
  } else {
    console.error('Unexpected error:', error);
  }
}
```

`APIError` クラスは組み込みの `Error` クラスを拡張してHTTPステータスコードを含めています。`handleError` 関数は `instanceof` を使ってAPIエラーとその他の例外を区別します。

### 要素の組み合わせ

1つのドキュメント内でさまざまな要素を組み合わせることができます：

- **リスト**: 列挙に
- **テーブル**: 構造化データに
- **コードブロック**: 技術的な例に
- **数式**: 数式の表現に：$f(x) = ax^2 + bx + c$
- **脚注**: 追加の文脈に<span class="footnote">この脚注は、本文の流れを妨げることなく補足情報を提供します。</span>
- **画像**: ビジュアルコンテンツに
- **引用**: 強調に

この柔軟性により、Vivliostyleはほぼあらゆる種類の出版物に対応できます。

## まとめ

これらの例は、Vivliostyle CLIとVFMの汎用性を示しています。技術文書、学術論文、クリエイティブ作品、ビジネスレポートなど、Markdownのシンプルさとプロフェッショナルな組版機能の組み合わせが出版物の優れた基盤を提供します。

さまざまな機能、テーマ、設定を試して、自分のニーズに最適なワークフローを見つけてください。
